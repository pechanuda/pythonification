# Lesson 01 Plan

## Theme

Python mindset and core collections.

This lesson is about getting comfortable with basic Python data structures and reading short Python code without treating it like compressed Java.

## Objectives

By the end of this lesson, you should be able to:

- choose between `list`, `dict`, `set`, and tuple
- read and write basic comprehensions
- use indexing, slicing, and unpacking
- transform small datasets in a Pythonic way
- understand simple scripts that process lists, filenames, and small records

## Time Budget

Target total: about 1 hour for the first pass.

Suggested split:

- 15-20 minutes reading the lecture
- 30-35 minutes solving the exercise
- 10 minutes reviewing and cleaning up your code

## Workflow

1. Read [lecture.md](/Users/michael.linka/Udemy/pythonification/lessons/lesson_01/lecture.md)
2. Work in [exercise.py](/Users/michael.linka/Udemy/pythonification/lessons/lesson_01/exercise.py)
3. Solve each section with normal loops first
4. Rewrite at least part of each section using comprehensions
5. Bring the code back for review

## What Success Looks Like

Success does not mean memorizing every syntax detail.

Success means:

- you can read the examples without freezing
- you can solve small collection-based tasks
- you can see when Python offers a simpler expression than a verbose loop
- you are starting to build pattern recognition for common syntax

## Common Mistakes to Avoid

- overusing classes for simple tasks
- writing overly ceremonial Java-shaped code
- trying to force streams onto every transformation
- using comprehensions when they hurt readability
- adding abstraction before it solves a real problem
